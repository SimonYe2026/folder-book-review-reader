# 开发者指南：架构与后续扩展

本文面向准备修改、嵌入或继续开发本项目的人。它描述当前架构、稳定边界和建议扩展顺序，不代表所有方向都已进入版本计划。

对应英文文件：`06-developer-guide.en.md`

## 当前稳定基线

```text
workspace.config.md
  -> build_reader.py 扫描、解析和打包
  -> 单个自包含 reader.html
  -> 浏览器阅读、筛选、批复和导出 review.md
```

Python 负责文件系统能力和生成过程；浏览器只处理已经打包的数据。阅读器不联网、不写回源文件，也不直接调用 AI。

## 代码中的主要边界

`build_reader.py` 当前同时包含四类职责：

1. 配置解析和路径安全检查；
2. 文件扫描、排序和章节数据生成；
3. Markdown/TXT 到安全 HTML 的转换；
4. HTML、CSS 和 JavaScript 应用的组装。

可选 DOCX 转换器位于 `tools/convert_docs.py`。它属于阅读器前置链路，不应让浏览器直接承担 Office 文件解析。

## 推荐扩展顺序

### 1. 先增加配置能力

界面文案、批复动作、批复目标、筛选规则和显示默认值，应优先通过配置扩展。配置可解决的问题，不必先增加新的运行时依赖。

### 2. 再扩展结构化数据

新增阅读或审阅能力时，先明确章节数据和批复数据需要哪些字段，再修改界面。文件路径、段落编号、引用原文和创建时间是批复定位的基础字段。

### 3. 将生成资源拆成源码模块

当 `build_reader.py` 继续增长时，推荐先把模板、CSS 和 JavaScript 拆成独立源码文件，构建时再内嵌到单个 HTML。这样能改善维护性，同时保持用户看到的一条命令和单文件产物不变。

建议结构：

```text
build_reader.py
reader_src/
  template.html
  reader.css
  reader.js
```

### 4. 最后评估前端框架

当前原生 `<dialog>` 足以承担批复弹窗，它已经提供模态层、焦点管理、Escape 关闭和表单语义。仅为了弹窗，不值得引入框架。

如果审阅板继续增加状态、组件和局部更新，可以评估 Vue 官方生产版 runtime。这里指官方发布的精简生产运行时，不是自行删改 Vue 源码。建议固定版本并本地分发或在构建时内嵌，避免 CDN 依赖，继续保持离线和自包含输出。

jQuery 只会简化部分 DOM 写法，不能解决响应式状态和组件边界问题，因此不是推荐演进方向。UI 组件库只有在确实需要大量一致的复杂控件时再评估。

## 值得继续探索的方向

- 更清晰的 Python 模块边界和模板资源拆分；
- 审阅板的组件化与更细粒度状态更新；
- 可插拔的输入转换器，但统一输出 Markdown/TXT 或明确的中间数据结构；
- 更完整的键盘和无障碍支持；
- 面向大型文件集合的搜索性能和按需渲染；
- 导出格式适配器，让不同 AI 或团队流程复用同一批复数据。

表格增强、小统计和更多视觉组件属于 UI 增量，优先级低于阅读、筛选、批复和导出主链路。

## 不建议直接扩展的方向

- 浏览器静默扫描或写回本地文件；
- 在默认阅读器中调用 AI API；
- 自动合并 AI 返回结果；
- 让浏览器直接解析 DOCX/PDF/EPUB；
- 根据字符画猜测并重绘复杂图表；
- 为单个控件引入完整前端工程链。

这些方向会改变安全边界或产品定位，适合独立分支、插件或另一个项目。

## 修改后的最低验证

每次修改至少运行：

```powershell
python tests\smoke_test.py
python build_reader.py workspace.config.md --dry-run
python build_reader.py workspace.config.md --overwrite
```

涉及前端交互时，还应人工检查：

- 搜索后上一篇/下一篇只在筛选结果中移动；
- 目录和审阅板折叠不挤掉正文；
- 中文输入法不会被审阅板重绘打断；
- 刷新后设置和批复能从 localStorage 恢复；
- 下载的 `review.md` 可重新打包并作为普通 Markdown 阅读；
- 不可信链接保持禁用，文本和内嵌数据不会变成可执行脚本。

## 兼容性原则

一个好的扩展应尽量保持：

- 一条 Python 命令完成构建；
- 最终仍可生成自包含 HTML；
- 默认离线、无遥测、无网络依赖；
- 源文件只读，批复旁路导出；
- 中文默认界面与英文国际化文件保持对应；
- 新能力有示例、文档和回归测试。

先守住这些性质，再增加技术层次。这个项目的价值不在于使用了多少框架，而在于它把本地文档、人工判断和 AI 后续处理连接成了一个可靠的小闭环。
