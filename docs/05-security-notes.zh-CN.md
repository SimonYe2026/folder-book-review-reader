# 安全说明

本文件说明本项目的安全模型、已有防护和仍然需要用户注意的风险。

对应英文文件：`05-security-notes.en.md`

## 基本判断

这个工具是本地优先、只读源文件的阅读器。

它做了多层防护，但不建议用户打包和打开来路不明的 Markdown/TXT 文件。

原因很简单：打包过程会把输入文件内容放进一个浏览器页面里。即使脚本已经尽量转义、禁用链接和限制图片，未知输入仍然可能带来显示异常、性能压力、误导性内容或未发现的边界问题。

## 已有防护

当前版本已经做了这些防护：

- 源文件只读，构建过程不修改源章节。
- `source_dir` 和 `output` 会从 `workspace_root` 解析，并且必须留在它里面。
- Markdown 链接默认不生成可点击 `<a href=...>`。
- 链接会显示为“未知链接，请回源文件查看”这类占位提示。
- 图片 URL 禁止 `javascript:` 等协议。
- `data:image/...` 仅允许常见 raster 图片类型：png、jpg、jpeg、gif、webp。
- 嵌入到 `<script>` 中的 JSON 会转义 `<`、`>`、`&`，避免 `</script>` 注入。
- 异常 `docx-table` 会回退为代码块，而不是中断构建。
- `dry-run` 可以在写出文件前查看读取范围。
- `overwrite: false` 时不会覆盖已有输出。

## MD、TXT 和 DOCX 的防护边界

这些防护并不是只针对 Markdown。

进入阅读器后的内容都会受到最终 HTML 打包层的保护：

- `.md`：走 Markdown 基础渲染。链接会被禁用，图片会被限制，嵌入 JSON 会被安全转义。
- `.txt`：不解释 Markdown，直接做 HTML 转义。即使文本里写了 `<script>`、`[link](javascript:...)` 或图片语法，也只会作为普通文本显示。
- `.docx`：不会被浏览器直接读取。它需要先通过可选转换器变成 `.md`，然后再进入 Markdown 阅读器的同一套防护。

需要注意的是：DOCX 转换器本身属于转换阶段，不等同于浏览器阅读阶段。

所以更准确的说法是：

```text
TXT 在阅读器内更保守；
MD 在阅读器内被限制渲染；
DOCX 转成 MD 后享受同一套阅读器防护，但转换器阶段仍不建议处理不可信 Office 文件。
```

## 仍然不建议处理不可信输入

请尽量只打包你自己生成、自己下载后已确认来源、或来自可信协作者的 Markdown/TXT。

不建议直接打包：

- 来路不明的 Markdown。
- 来路不明的 TXT。
- 未审查的 AI 批量生成文件。
- 来路不明的 DOCX 或其他 Office 文档。
- 从网页、邮件、聊天工具里直接复制来的大块内容。
- 包含大量超长行、超大 base64、异常嵌套表格的文件。
- 你不准备暴露给浏览器页面的数据。

这些输入即使不一定造成脚本执行，也可能带来：

- 页面体积过大。
- 浏览器卡顿。
- 内容误导。
- 搜索和批复定位困难。
- 未知渲染边界。

## 生成 HTML 的敏感性

生成的 `reader.html` 是内容打包文件，不是空壳索引。

如果源文件包含敏感信息，生成的 HTML 也会包含这些信息。

不要把包含敏感内容的 HTML 上传到公开平台，也不要直接发给不该看到这些内容的人。

## localStorage 说明

阅读设置和批复会保存到浏览器 `localStorage`。

这意味着：

- 刷新页面后批复可以恢复。
- 同一浏览器、同一页面来源下可能保留历史批复。
- 如果使用本地 HTTP 服务，状态会按该本地地址保存。

如果内容敏感，发布或演示前请清空批复，或换一个干净浏览器环境。

## 本地服务说明

`serve_reader.py` 默认绑定：

```text
127.0.0.1
```

这是本机访问地址。

不建议把服务绑定到公网或局域网可访问地址。因为本地服务会把项目目录作为静态文件根目录提供访问。

## 转换器说明

可选 DOCX 转换器会读取 `.docx` 并生成 Markdown。

转换器不修改源文件，但仍不建议处理来路不明的 Office 文档。未知文档可能非常大、结构异常，或包含转换器尚未覆盖的复杂内容。

## 推荐使用方式

更稳妥的流程是：

```text
确认来源可信
  -> dry-run 查看读取范围
  -> 构建 reader.html
  -> 本地打开阅读
  -> 导出 review.md
```

如果只是想查看陌生 Markdown 内容，建议先用普通文本编辑器打开检查，再决定是否打包进阅读器。
