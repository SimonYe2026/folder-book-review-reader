# 用户手册

本文件面向普通使用者，说明阅读器界面、快捷键、审阅板和导出流程。

对应英文文件：`04-user-manual.en.md`

## 生成阅读器

在项目目录中运行：

```powershell
python build_reader.py workspace.config.md
```

生成文件：

```text
output/reader.html
```

也可以启动本地预览服务：

```powershell
python serve_reader.py
```

默认地址：

```text
http://127.0.0.1:8766/output/reader.html
```

## 界面结构

阅读器主要分为三栏：

- 左侧：目录栏。
- 中间：正文阅读区。
- 右侧：审阅板。

目录栏和审阅板都可以折叠。左右栏宽度可以拖拽调整，并会保存到浏览器本地状态。

## 顶部工具栏

顶部工具栏包含：

- 搜索框。
- 文件类型筛选。
- 正序/倒序切换。
- 字体大小调整。
- 阅读宽度调整。
- 深色/浅色模式切换。
- 恢复默认布局按钮。
- 目录栏折叠按钮。
- 审阅板折叠按钮。

搜索会匹配：

- 标题。
- 文件名。
- 相对路径。
- 正文内容。

搜索后的结果会形成临时书单。上一篇和下一篇只在当前筛选结果中移动。

## 快捷键

当前快捷键：

| 快捷键 | 功能 |
| --- | --- |
| `A` | 上一篇 |
| `←` | 上一篇 |
| `D` | 下一篇 |
| `→` | 下一篇 |
| `R` | 打开审阅窗口 |

当光标位于输入框、下拉框或文本框中时，快捷键不会触发翻页或打开审阅窗口。

在 Microsoft Edge 等浏览器中，选中文本后浏览器可能显示自己的迷你菜单。这个菜单有时会挡住页面或截获按键，导致 `R` 没有打开审阅窗口。Firefox 中通常不会出现这个问题。遇到这种情况时，直接点击页面中的浮动批复按钮，或点击段落旁的批复按钮。

## 阅读设置

可以调整：

- 字体大小。
- 阅读宽度。
- 深色/浅色模式。
- 左侧目录栏宽度。
- 右侧审阅板宽度。
- 目录栏和审阅板是否折叠。

这些设置会保存在浏览器 `localStorage` 中。刷新页面后仍会恢复。

恢复默认按钮会重置阅读布局相关设置，但不会删除批复。

## 添加批复

可以用两种方式添加批复：

- 选中文本后点击浮动批复按钮，或按 `R`。
- 点击段落旁边的批复按钮。

如果选中了文本，审阅窗口会优先引用选中文本。

如果没有选中文本，审阅窗口会引用当前段落。

审阅窗口包含：

- 引用原文。
- 动作。
- 目标。
- 用户批复说明。

动作和目标来自配置文件，可以按自己的工作流修改。

## 审阅板

审阅板会显示当前所有批复。

支持：

- 编辑动作。
- 编辑目标。
- 编辑用户批复说明。
- 删除单条批复。
- 清空所有批复。
- 复制 `review.md`。
- 下载 `review.md`。

编辑审阅板中的批复时，引用原文不会被修改。

## 导出的 review.md

导出的 `review.md` 按文件分组。

每条批复通常包含：

- 文件路径。
- 标题。
- 段落位置。
- 动作。
- 目标。
- 时间。
- 引用原文。
- 用户批复。

这个文件可以交给 AI，也可以作为人工修改清单。

推荐用法是：

```text
复制或下载 review.md
  -> 粘贴给 AI，或作为文件交给 AI
  -> 要求 AI 根据批复提出修改建议或生成新版本
  -> 用户确认后，再决定是否修改源文件
```

阅读器不会替用户自动执行这些修改。

完整闭环可以是：

```text
任意来源文件
  -> 打包 reader.html
  -> 人工阅读和审批
  -> 导出 review.md
  -> AI 根据批复修改源文件或生成新版本
  -> 再次打包 reader.html
  -> 人工验收
```

## 重新打包 review.md

下载后的 `review.md` 可以放入一个文件夹，然后再次用阅读器打包。

这适合：

- 复读批复。
- 归档批复。
- 二次审阅批复。
- 把多个批复文件合并成一个可搜索阅读器。

这条链路不承诺结构可逆，只承诺作为普通 Markdown 可读、可搜索、可继续审阅。

## 链接和图片

Markdown 链接默认不会被启用。

阅读器会显示：

```text
（未知链接，请回源文件查看）
```

这样做是为了避免外部或 AI 生成的 Markdown 在浏览器中激活不可信链接。

图片支持基础显示。转换器生成的 data URL 图片可以进入阅读器，但复杂文档中的图片提取质量取决于转换器。

## 常见问题

### 新增源文件后页面为什么没变化？

需要重新运行构建命令。

阅读器 HTML 是一次性打包产物，不会自动监听文件夹。

### 为什么浏览器不能直接读取当前文件夹？

浏览器出于安全限制，不能静默扫描本地目录。

当前设计是：

```text
Python 读取本地文件
  -> HTML 负责阅读和交互
```

### 为什么不直接修改源文件？

为了安全和可控。

阅读器只生成旁路批复文件。是否修改源文件，由用户或后续 AI 流程决定。
