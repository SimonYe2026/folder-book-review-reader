# 用户手册

本文件面向普通使用者，说明阅读器界面、快捷键、审阅板和导出流程。

对应英文文件：`04-user-manual.en.md`

## 生成阅读器

在项目目录中运行：

```powershell
python build_reader.py
```

生成文件：

```text
output/reader.html
```

也可以启动本地预览服务：

```powershell
python serve_reader.py
```

默认地址：

```text
http://127.0.0.1:8766/output/reader.html
```

## 审阅者最低条件

构建者和中心侧负责运行 Python 打包、回收归并、AI 处理、源文件修改与最终采纳。审阅者只负责阅读、判断、批复和导出回执；不需要安装 Python、不需要项目依赖、不需要接触源目录，也不需要懂 AI 或提示词。

收到 `reader.html` 后，用任意现代浏览器打开即可阅读、筛选、加入批复并导出 `review.md`。批复会暂存在当前浏览器，完成一轮审阅后应导出 `review.md` 交回协调者或后续维护流程。

## 界面结构

阅读器主要分为三栏：

- 左侧：目录和书签栏。
- 中间：正文阅读区。
- 右侧：审阅板。

目录栏和审阅板都可以折叠。左右栏宽度可以拖拽调整，并会保存到浏览器本地状态。

## 顶部工具栏

顶部工具栏包含：

- 搜索框。
- 文件类型筛选。
- 正序/倒序切换。
- 字体大小调整。
- 阅读宽度调整。
- 深色/浅色模式切换。
- 恢复默认布局按钮。
- 添加或移除当前位置书签按钮。
- 目录栏折叠按钮。
- 审阅板折叠按钮。

搜索会匹配：

- 标题。
- 文件名。
- 相对路径。
- 正文内容。

搜索后的结果会形成临时书单。上一篇和下一篇只在当前筛选结果中移动。

## 快捷键

当前快捷键：

| 快捷键 | 功能 |
| --- | --- |
| `A` | 上一篇 |
| `←` | 上一篇 |
| `D` | 下一篇 |
| `→` | 下一篇 |
| `B` | 添加或移除当前位置书签 |
| `R` | 打开审阅窗口 |

当光标位于输入框、下拉框或文本框中时，快捷键不会触发翻页、书签或打开审阅窗口。

在 Microsoft Edge 等浏览器中，选中文本后浏览器可能显示自己的迷你菜单。这个菜单有时会挡住页面或截获按键，导致 `R` 没有打开审阅窗口。Firefox 中通常不会出现这个问题。遇到这种情况时，直接点击页面中的浮动批复按钮，或点击内容块旁的批复按钮。

## 阅读设置

可以调整：

- 字体大小。
- 阅读宽度。
- 深色/浅色模式。
- 左侧目录栏宽度。
- 右侧审阅板宽度。
- 目录栏和审阅板是否折叠。

这些设置会保存在浏览器 `localStorage` 中。刷新页面后仍会恢复。

恢复默认按钮会重置阅读布局相关设置，但不会删除批复或书签。

## 书签

书签按当前位置的内容块保存，不是整篇文件书签。标题、普通段落、列表项、引用块、代码块、Markdown 表格、转换后的 DOCX 表格、TXT 行和 CSV 行都可以添加书签。

- 点击页面底部固定导航栏中间的“添加书签”保存当前位置；再次点击同一位置会移除书签。也可按 `B`。
- 直接点击某个内容块后添加书签，会优先保存这一内容块；没有明确点击内容块时，才按正文可视区域中心判断当前位置。
- 滚动正文后，阅读器会取消先前内容块的书签定位，避免在新位置误移除旧书签。
- 未选中文本、也未通过内容块旁按钮明确指定位置时，书签和批复会使用正文可视区域中心最近的内容块。把希望处理的位置滚动到正文中部，再按 `B` 或 `R` 即可；CSV 仍优先使用当前激活行。
- 左侧栏可用“目录 / 书签”按钮切换视图。这个切换只改变左侧栏显示内容，不会改变搜索条件、文件类型筛选或已有书签。
- 点击书签会跳回对应文件和内容块，并短暂高亮目标位置。
- 每条书签右侧都有“删除”按钮；书签视图右上角可“清空书签”，需要确认后才会删除全部书签。
- 如果当前搜索或文件类型筛选隐藏了目标文件，阅读器会自动清空这些筛选后再跳转。

书签只保存在当前浏览器的 `localStorage` 中，不会写入源文件，也不会进入导出的 `review.md`。

## 定位与复查

每条批复都会记录文件路径、内容块位置、引用原文、动作、目标和批复说明。选中文本或点击内容块旁的“批复”时，审阅范围标为精确对象；未选中文本而按 `R` 或点击浮动批复按钮时，审阅范围标为“阅读视口”，核心引用只用于定位。审阅板和导出的 `review.md` 会提供对应范围的附近上下文，并标明内容块范围，例如“附近内容块 2-4”。这让后续人工或 AI 能结合批复说明判断实际处理对象，而不必猜测整章意图。点击审阅板中的批复项，阅读器会跳回对应文件并短暂高亮核心目标位置。

当前可定位的内容包括：标题、普通段落、列表项、引用块、代码块、Markdown 表格、转换后的 DOCX 表格、TXT 行和 CSV 行。CSV 单元格选区会跳回其起始行。若搜索或文件类型筛选隐藏了目标文件，阅读器会先恢复目标文件可见，再执行跳转。

书签适合标记稍后要复看的内容位置；批复适合记录需要交回中心处理的判断。两者都可以从左侧书签栏或右侧审阅板回到原文。

## 添加批复

可以用两种方式添加批复：

- 选中文本后点击浮动批复按钮，或按 `R`。
- 点击段落旁边的批复按钮。

如果选中了文本，审阅窗口会优先引用选中文本。

如果没有选中文本，审阅窗口会引用当前段落。

## CSV 表格

CSV 文件会作为表格型章节显示。

- 第一行作为表头。
- 数据行显示为 HTML 表格行。
- 表格可以横向滚动。
- 搜索可以命中表头和单元格内容。
- 每个数据行都可以加入批复。

给 CSV 加批复时：

- 如果选中了单元格或行内文本，优先引用选中文本。
- 如果选中的是 CSV 单元格区域，位置会尽量记录行号和列号范围。
- 如果没有选中文本，会引用当前行。
- 导出的 `review.md` 会把位置显示为“第 N 行”。

阅读器不会在浏览器里编辑 CSV，也不会写回 CSV 源文件。修改 CSV 仍然发生在阅读器外，由用户或后续 AI 流程完成。

审阅窗口包含：

- 引用原文。
- 动作。
- 目标。
- 用户批复说明。

动作和目标来自配置文件，可以按自己的工作流修改。

## 审阅板

审阅板会显示当前所有批复。

支持：

- 点击批复项，跳回对应文件和标题、段落、列表项、引用块、代码块、Markdown 表格、转换后的 DOCX 表格、TXT 行或 CSV 行。
- 编辑动作。
- 编辑目标。
- 编辑用户批复说明。
- 删除单条批复。
- 清空所有批复。
- 复制 `review.md`。
- 下载 `review.md`。

编辑审阅板中的批复时，引用原文不会被修改。

## 导出的 review.md

导出的 `review.md` 按文件分组。

每条批复通常包含：

- 文件路径。
- 标题。
- 段落位置或 CSV 行位置。
- 动作。
- 目标。
- 时间。
- 引用原文。
- 用户批复。

这个文件可以交给 AI，也可以作为人工修改清单。

推荐用法是：

```text
复制或下载 review.md
  -> 粘贴给 AI，或作为文件交给 AI
  -> 要求 AI 根据批复提出修改建议或生成新版本
  -> 用户确认后，再决定是否修改源文件
```

阅读器不会替用户自动执行这些修改。

完整闭环可以是：

```text
任意来源文件
  -> 打包 reader.html
  -> 人工阅读和审批
  -> 导出 review.md
  -> AI 根据批复修改源文件或生成新版本
  -> 再次打包 reader.html
  -> 人工验收
```

## 重新打包 review.md

下载后的 `review.md` 可以放入一个文件夹，然后再次用阅读器打包。

这适合：

- 复读批复。
- 归档批复。
- 二次审阅批复。
- 把多个批复文件合并成一个可搜索阅读器。

这条链路不承诺结构可逆，只承诺作为普通 Markdown 可读、可搜索、可继续审阅。

## 链接和图片

Markdown 链接默认不会被启用。

阅读器会显示：

```text
（未知链接，请回源文件查看）
```

这样做是为了避免外部或 AI 生成的 Markdown 在浏览器中激活不可信链接。

图片支持基础显示。转换器生成的 data URL 图片可以进入阅读器，但复杂文档中的图片提取质量取决于转换器。

## 常见问题

### 新增源文件后页面为什么没变化？

需要重新运行构建命令。

阅读器 HTML 是一次性打包产物，不会自动监听文件夹。

### 为什么浏览器不能直接读取当前文件夹？

浏览器出于安全限制，不能静默扫描本地目录。

当前设计是：

```text
Python 读取本地文件
  -> HTML 负责阅读和交互
```

### 为什么不直接修改源文件？

为了安全和可控。

阅读器只生成旁路批复文件。是否修改源文件，由用户或后续 AI 流程决定。
