# 项目状态说明

本文件说明 Local Markdown/TXT/CSV Review Reader 当前版本的定位、已完成能力、边界和后续方向。

对应英文文件：`01-project-status.en.md`

## 当前定位

这个项目是一个本地优先的 Markdown/TXT/CSV 审阅阅读器。

它的核心用途是：

- 把一个文件夹里的 `.md` / `.txt` / `.csv` 文件打包成一本可阅读的单文件 HTML。
- 在浏览器中搜索、筛选、连续翻篇。
- 选中文本或段落，生成面向 AI 或人工后续处理的批复记录。
- 导出按文件分组的 `review.md`。
- 将大量文件作为一个可浏览、可定位批复、可回收回执的审阅交接包分发。
- 让不具备 AI、Python、代码仓库或源目录操作能力的审阅者，仅用现代浏览器参与批复。

它不是：

- Markdown 编辑器。
- 在线 AI 客户端。
- 文件管理器。
- 知识库系统。
- 文档站生成器。
- 多格式万能转换器。

## 当前稳定主链路

当前主链路已经闭合：

```text
本地 Markdown/TXT/CSV 文件夹
  -> build_reader.py
  -> output/reader.html
  -> 浏览器阅读、筛选、批复
  -> 导出 review.md
```

重新打包 `review.md` 也已经验证可行：

```text
review.md
  -> 作为普通 Markdown 文件重新进入阅读器
  -> 用于复读、归档、二次审阅
```

## 已完成能力

- 递归或非递归读取 `.md` / `.txt` / `.csv`。
- 自然排序、倒序、按修改时间排序。
- 保留相对路径、文件名、扩展名、字数、修改时间。
- 全文搜索标题、路径、文件名和正文。
- 筛选结果即临时书单，上一篇/下一篇基于当前筛选结果。
- 字体大小、阅读宽度、深色模式、左右栏宽度持久化。
- 目录栏和审阅栏可折叠。
- 支持段落级批复和选中文本批复。
- 审阅板支持编辑、删除、清空、复制、下载。
- `review.md` 按文件分组导出。
- Markdown 表格、代码块、引用、列表、图片基础渲染。
- CSV 表格展示、全文搜索和按行批复。
- 链接默认禁用，显示来源提示，避免激活不可信链接。
- `dry-run`、输出不覆盖、清晰错误提示。
- 可选 `.docx -> .md` 转换器。
- 根目录和 `config/` 两种可迁移配置模板，方便嵌入到其他 Markdown/TXT/CSV 项目。
- `tests/html_quality_check.py` 静态检查生成 HTML 的页面结构、章节数据、审阅钩子、导航钩子和安全边界。

## 安全边界

这个项目刻意保持保守：

- 不联网。
- 不上传文件。
- 不调用 AI API。
- 不从浏览器写回源文件。
- 不删除、移动、重命名源文件。
- 不启用 Markdown 链接。
- 不在浏览器中解析 Office/PDF/EPUB。

生成的 HTML 会包含源文件正文。如果源文件有敏感内容，不要公开分享生成的 HTML。

## 当前不承诺事项

- 不承诺复杂 DOCX 版式完全还原。
- 不承诺 `.doc` / `.rtf` / `.pdf` / `.epub` 支持。
- 不承诺图表、流程图、字符画自动重绘。
- 不承诺 AI 返回结果合并。
- 不提供多人共享编辑、实时状态同步或云同步。

当前版本支持多方审阅协作：协调者为不同批次或展示范围打包不同的 `reader.html`，多人分别审查后回收多份 `review.md`，再由中心或后续 AI 汇总意见并决定修改。协作单位是批复回执，不是多人同时修改源文件。

这让“压缩包 + 零散聊天反馈”的大文件审阅方式，变成“审阅包 + 带坐标的结构化回执”。几位审阅者或小团队可直接使用；人数、批次和轮次继续增加时，认领、命名、截止、集中回收和采纳决策需要配套的项目约定。它们是协作扩展的组织层，不否定当前 reader/review 链路的协作价值。

## 适合自定义的方向

这个项目适合被后来者按自己的工作流修改：

- 改 UI 文案和语言。
- 改批复动作和目标。
- 改阅读布局宽度。
- 改 `review.md` 导出格式。
- 增加新的文件转换器。
- 增加新的 Markdown 渲染规则。
- 基于模板增加项目内部的固定配置。
- 增加更高层级的浏览器动态交互测试脚本，用于模拟搜索、翻页、目录点击、审阅弹窗和批复编辑等真实点击流。

建议自定义时优先保持三条边界：

- 源文件只读。
- 浏览器不越权。
- 生成内容可被普通 AI 或人类继续阅读。

测试也按层级管理：静态页面质量检查用于快速确认 HTML 产物结构完整；浏览器动态交互测试属于下一层脚本，不应和静态检查混成一个重型入口。

## 当前结论

当前版本已经达到可开源的基础状态。

它不是大平台，而是一个小而清晰的工具：让本地 Markdown/TXT/CSV 文件夹可以被连续阅读、筛选、批复，并把批复变成 AI 友好的后续任务。
